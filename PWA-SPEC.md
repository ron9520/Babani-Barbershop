# 📱 מספרת בבאני — אפיון PWA

## מטרת המסמך
אפיון מלא ל-Progressive Web App עבור מספרת בבאני — פאנל ניהול לחיים הספר + דף נחיתה להזמנת תורים ללקוחות.

---

## 1. ארכיטקטורה כללית

### מבנה האפליקציה
```
PWA
├── /admin          ← פאנל ניהול (חיים בלבד, מוגן PIN)
│   ├── דשבורד ראשי + סטטיסטיקות
│   ├── תצוגת יום / תצוגת שבוע
│   ├── ניהול שעות פעילות
│   └── ניהול מחירון
│
├── /                ← דף נחיתה ללקוחות (פומבי)
│   └── הזמנת תור עצמאית
│
└── Service Worker   ← Push Notifications + קאשינג
```

### סטאק טכנולוגי
- **Frontend**: React (Single Page App) עם Tailwind CSS
- **Backend**: Express.js (מרחיב את השרת הקיים)
- **DB**: Firebase Firestore (כבר קיים)
- **יומן**: Google Calendar API (כבר מחובר)
- **PWA**: Service Worker + Web App Manifest
- **Push**: Firebase Cloud Messaging (FCM) או Web Push API
- **Hosting**: Render (כבר מוגדר)

### למה React ולא HTML סטטי?
- ניווט בין מסכים בלי טעינה מחדש (SPA)
- ניהול state מורכב (תצוגות, פילטרים, טפסים)
- קומפוננטות reusable (כרטיס תור, בורר תאריכים)
- ביצועים טובים עם virtual DOM
- קל להוסיף פיצ'רים בעתיד

### 2.3 תהליך הזמנת תור (מהדף)

**שלב 1 — בחירת שירות**
- רשימת כרטיסים עם שם + מחיר + משך
- לחיצה על כרטיס = בחירה

**שלב 2 — בחירת תאריך**
- Calendar picker ויזואלי
- ימים סגורים (שבת) מסומנים באפור
- ימים עם זמינות מלאה מודגשים
- חלון הזמנה: עד 14 יום קדימה (לפי config)

**שלב 3 — בחירת שעה**
- רשימת כפתורי שעות פנויות (grid)
- שעות תפוסות לא מוצגות
- הצגה בפורמט 24H ישראלי (14:30)

**שלב 4 — פרטי לקוח**
- שם מלא (חובה)
- מספר טלפון (חובה, ולידציה ישראלית)

**שלב 5 — סיכום + אישור**
- הצגת כל הפרטים
- כפתור "אישור"
- אנימציית הצלחה + הודעת WhatsApp אוטומטית

### 2.4 API קיים (כבר עובד!)
- `GET /api/services` — רשימת שירותים
- `GET /api/slots?date=YYYY-MM-DD` — שעות פנויות
- `POST /api/book` — הזמנת תור
- לא צריך לבנות backend חדש — רק frontend!

## 3. פאנל ניהול — חיים (נתיב: `/admin`)

### 3.1 אבטחה — כניסה עם PIN

```
┌─────────────────────────────┐
│                              │
│       🔒 פאנל ניהול          │
│                              │
│     הזן קוד כניסה:          │
│     ┌──┐ ┌──┐ ┌──┐ ┌──┐    │
│     │  │ │  │ │  │ │  │    │
│     └──┘ └──┘ └──┘ └──┘    │
│                              │
│     [ 🔓 כניסה ]            │
│                              │
└─────────────────────────────┘
```

- קוד PIN בן 4 ספרות
- PIN שמור ב-environment variable (`ADMIN_PIN`)
- בצד הלקוח: שמירת token ב-sessionStorage (לא localStorage)
- ב-backend: אימות PIN מחזיר JWT token עם תוקף 24 שעות
- כל endpoint של admin דורש header: `Authorization: Bearer <token>`
- 3 ניסיונות כושלים = נעילה ל-5 דקות

### 3.2 ניווט ראשי (Bottom Navigation)

```
┌───────┬───────┬──────┬───────┐
│ 📅    │  📊   │ ⚙️   │  💰   │
│תורים  │סטטיס. │הגדרות│מחירון │
└───────┴───────┴──────┴───────┘
```

4 טאבים בתפריט תחתון (מותאם לאייפון):
1. **תורים** — תצוגת יום/שבוע
2. **סטטיסטיקות** — נתונים ודוחות
3. **הגדרות** — שעות פעילות + הגדרות כלליות
4. **מחירון** — ניהול שירותים ומחירים

**שני סוגי שינויים:**

1. **שינוי קבוע** — עדכון שעות הפעילות הרגילות (config)
   - משנה את ברירת המחדל לאותו יום בשבוע
   - משפיע על כל השבועות מעכשיו

2. **שינוי חד-פעמי (Override)** — לתאריך ספציפי
   - "מחר אני עובד רק עד 15:00"
   - "ביום רביעי אני סגור"
   - נשמר ב-Firestore באוסף `schedule_overrides`
   - מבנה:
   ```json
   {
     "date": "2026-03-25",
     "open": "09:00",
     "close": "15:00",
     "closed": false,
     "reason": "סיום מוקדם"
   }
   ```
   - ⚠️ **חשוב**: הבוט + דף הזמנה חייבים לבדוק overrides לפני שמציעים slots!

## 4. Push Notifications

### 4.1 מתי נשלחות התראות לחיים?

| אירוע | הודעה | תזמון |
|--------|--------|--------|
| תור חדש נקבע | "📅 תור חדש: דוד כהן, תספורת, מחר 10:00" | מיידי |
| תור בוטל | "❌ תור בוטל: דוד כהן, מחר 10:00" | מיידי |
| תזכורת בוקר | "☀️ יש לך 5 תורים היום. הראשון ב-09:00" | 07:30 כל בוקר |
| לקוח לא הגיע | "👻 דוד כהן לא הגיע לתור של 10:00" | 15 דקות אחרי התור |

### 4.2 מימוש טכני
- **Firebase Cloud Messaging (FCM)** — עובד גם ב-PWA על iOS (מ-iOS 16.4+)
- Service Worker רושם את המכשיר ושומר token ב-Firestore
- שרת שולח push דרך FCM Admin SDK (כבר יש firebase-admin)
- fallback: הודעת WhatsApp (כבר קיים)

### 4.3 הערה לגבי iOS
- Push ב-PWA על iOS נתמך מ-**iOS 16.4** ומעלה
- חובה להתקין את ה-PWA ל-Home Screen (לא עובד מ-Safari רגיל)
- האייפון של חיים צריך להיות iOS 16.4+

### 5.2 Service Worker — אסטרטגיית קאשינג
- **App Shell** (HTML/CSS/JS): Cache First — טעינה מהירה
- **API calls** (תורים, slots): Network First — תמיד עדכני
- **אייקונים/תמונות**: Cache First with expiration
- **Offline**: הצגת הודעה "אין חיבור, מציג נתונים אחרונים"

### 5.3 התקנה על אייפון
1. חיים פותח את `https://babani.render.com/admin` ב-Safari
2. לוחץ על "שתף" → "הוסף למסך הבית"
3. האפליקציה מופיעה כאייקון על המסך
4. נפתחת ב-standalone mode (בלי Safari bar)

```
GET /api/admin/appointments?from=2026-03-24&to=2026-03-29

PATCH /api/admin/appointments/:id/status
  Body: { status: "completed" | "no_show" | "cancelled" }
  → עדכון סטטוס תור

DELETE /api/admin/appointments/:id
  → ביטול תור (מוחק גם מ-Calendar)
```

### 6.3 שעות פעילות
```
GET /api/admin/schedule
  → שעות פעילות רגילות + overrides קרובים

PUT /api/admin/schedule/default
  Body: { day: "monday", open: "09:00", close: "15:00" }
  → עדכון שעות קבועות

POST /api/admin/schedule/override
  Body: { date: "2026-03-25", open: "09:00", close: "15:00", reason: "..." }
  → שינוי חד-פעמי

DELETE /api/admin/schedule/override/:date
  → מחיקת שינוי חד-פעמי
```

```
  → מחיקת שירות
```
