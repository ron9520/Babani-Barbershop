# 📱 אפיון — פורטל לקוחות (Customer Portal)
## תוספת ל-PWA מספרת בבאני

---

## 1. סקירה כללית

### מטרה
לאפשר ללקוחות שכבר הזמינו תור (דרך WhatsApp או דף הנחיתה) לראות את התורים שלהם, לבטל תור, ולקבל התראות ישירות לנייד — בלי להיכנס ל-WhatsApp.

### עקרון מרכזי
**אין הרשמה.** הלקוח מזדהה רק עם מספר הטלפון שלו. אם יש לו תורים במערכת — הוא רואה אותם. שמו כבר ידוע מהתור הקודם.

## 2. Push Notifications ללקוחות

### שאלה חשובה: iOS + PWA

| פלטפורמה | תמיכה ב-Push | תנאי |
|-----------|-------------|------|
| אייפון iOS 16.4+ | ✅ כן | חייב להיות מותקן כ-PWA (Home Screen) |
| אייפון iOS < 16.4 | ❌ לא | — |
| אנדרואיד | ✅ כן | גם מהדפדפן |
| מחשב | ✅ כן | גם מהדפדפן |

### ⚠️ חשוב לדעת
- על **אייפון** — Push עובד **רק** אחרי שהלקוח מוסיף את האפליקציה ל-Home Screen
- כדאי להציג הנחיה ויזואלית: `"הוסף לדף הבית כדי לקבל תזכורות לתור"`
- **Fallback**: אם Push לא זמין → ממשיכים לשלוח תזכורת WhatsApp כרגיל

### מתי הלקוח מקבל Push?

| אירוע | תוכן ההתראה | תזמון |
|--------|------------|--------|
| אישור תור | "✅ התור שלך נקבע! תספורת ב-25/03 בשעה 10:00" | מיידי |
| תזכורת יום לפני | "⏰ תזכורת: תור מחר ב-10:00 במספרת בבאני" | 08:00 בבוקר |
| ביטול ע"י ספר | "❌ התור שלך ל-25/03 בוטל. לחץ לקביעת תור חדש" | מיידי |
| תור בעוד שעה | "💈 בעוד שעה! אל תשכח את התור שלך ב-10:00" | שעה לפני |

### רישום Push
- בעת כניסה לפורטל הלקוח (אחרי אימות טלפון) → בקשת הרשאת Push
- FCM token נשמר ב-Firestore תחת `customer_push_tokens/{phone}`
- token מקושר למספר הטלפון (לא ל-appointment ספציפי)

## 3. זרימת כניסה (Authentication)

### עיקרון: OTP דרך WhatsApp

**למה WhatsApp ולא SMS?**
- הלקוחות כבר עובדים עם המספרה דרך WhatsApp
- אין צורך ב-Twilio SMS (חוסך כסף)
- מוכר ונוח יותר ללקוחות ישראלים

### זרימת כניסה מלאה:

```
שלב 1 — הזנת טלפון
┌─────────────────────────────┐
│   📱 כניסה לתורים שלי       │
│                              │
│   הזן מספר טלפון:           │
│   ┌────────────────────┐    │
│   │ 05X-XXX-XXXX       │    │
│   └────────────────────┘    │
│                              │
│   [ שלח קוד אימות בוואטסאפ ]│
│                              │
│   _ישלח קוד בן 4 ספרות_     │
└─────────────────────────────┘

שלב 2 — קוד OTP
┌─────────────────────────────┐
│   שלחנו קוד ל-052-XXX-X554 │
│                              │
│   ┌──┐ ┌──┐ ┌──┐ ┌──┐      │
│   │  │ │  │ │  │ │  │      │
│   └──┘ └──┘ └──┘ └──┘      │
│                              │
│   [ אמת ]                   │
│                              │
│   שלח שוב (00:45)           │
└─────────────────────────────┘

שלב 3 — כניסה מוצלחת
┌─────────────────────────────┐
│  ✅ שלום, דוד!               │  ← שם נשלף אוטומטית מהתור
│  מוצגים התורים שלך           │
└─────────────────────────────┘
```

### לוגיקת OTP:
- קוד 4 ספרות אקראי
- נשמר ב-Firestore: `otp_codes/{phone}` עם תוקף **5 דקות**
- נשלח ללקוח דרך Green-API (WhatsApp) — אותה מערכת קיימת
- 3 ניסיונות כושלים → נעילה 10 דקות
- אחרי אימות מוצלח → JWT token עם תוקף **30 יום**
- token נשמר ב-localStorage של הנייד

### מה קורה אם הלקוח חדש (אין לו תורים)?
- כניסה תצליח, אבל מסך "התורים שלי" יציג: `"לא נמצאו תורים. רוצה לקבוע?"` + כפתור לדף הזמנה

```
┌─────────────────────────────┐
│  ⚠️ ביטול תור               │
│                              │
│  תור ב-25/03 בשעה 10:00     │

│  [ ← לא, חזור ]             │
└─────────────────────────────┘
```

**אחרי ביטול:**
- מחיקה מ-Google Calendar
- עדכון סטטוס ב-Firestore ל-`cancelled` + `cancelledBy: 'customer'`
- הודעת WhatsApp לחיים (ספר) על הביטול
- Push notification לאישור ללקוח: "✅ התור שלך בוטל בהצלחה"

**מגבלות ביטול:**
- ניתן לבטל עד **3 שעות לפני** התור
- מתחת ל-3 שעות → הודעה: "לביטול דחוף, צור קשר ישירות: [כפתור WhatsApp]"

### מתי להציג?
- כשנכנסים לדף הנחיתה **מ-Safari** (לא כ-PWA מותקן)
- אחרי שהלקוח קובע תור בפעם הראשונה

### מה להציג?

```

│  💡 טיפ: קבל תזכורות!      │
│                              │
```

## 6. API חדשים נדרשים

### 6.1 אימות לקוח
```
POST /api/customer/send-otp
  Body: { phone: "0523385554" }
  → שולח OTP ב-WhatsApp
  → Response: { success: true, expiresIn: 300 }

POST /api/customer/verify-otp
  Body: { phone: "0523385554", otp: "4821" }
  → Response: { token: "jwt...", name: "דוד כהן" }
    (שם נשלף מהתור האחרון של המספר)
```

### 6.2 תורים של לקוח
```
GET /api/customer/appointments
  Header: Authorization: Bearer <customer-jwt>
  → מחזיר תורים קרובים + היסטוריה לפי מספר הטלפון

DELETE /api/customer/appointments/:id
  Header: Authorization: Bearer <customer-jwt>
  → ביטול תור (בדיקה שהתור שייך למספר הטלפון)
  → בדיקה שלא מאוחר מדי (3 שעות)
  → מחיקה מ-Calendar + עדכון Firestore + הודעה לחיים
```

### 6.3 Push Notifications
```
POST /api/customer/push/register
  Header: Authorization: Bearer <customer-jwt>
  Body: { token: "fcm-device-token" }
  → שמירה ב-customer_push_tokens/{phone}

DELETE /api/customer/push/unregister
  Header: Authorization: Bearer <customer-jwt>
  → מחיקת token
```

## 7. מבנה Firestore — תוספות

```
otp_codes/
  └── {phone}  →  { otp, createdAt, attempts, lockedUntil }

customer_tokens/
  └── {phone}  →  { jwtTokenHash, issuedAt, lastSeen }

customer_push_tokens/
  └── {phone}  →  { token, platform: "ios"|"android", registeredAt }
```

**שינוי ב-appointments:**
```js
{
  ...existing,
  cancelledBy: "customer" | "barber" | null   // ← חדש
}
```

## 8. אבטחה

| איום | הגנה |
|------|------|
| מישהו מנסה לנחש OTP | נעילה אחרי 3 ניסיונות, OTP תוקף 5 דק' |
| גישה לתורים של מישהו אחר | JWT כולל phone, בדיקה ש-appointment.phone === token.phone |
| ביטול תור של לקוח אחר | אותה בדיקה לעיל |
| Token גנוב | תוקף 30 יום, logout מנקה מ-localStorage |
| Brute force על OTP | rate limiting: max 3 בקשות OTP לשעה למספר |

## 9. שינויים בקוד קיים

### whatsappService.js
פונקציה חדשה: `sendOTP(phone, code)`:
```
✂️ *מספרת בבאני — קוד אימות*

קוד הכניסה שלך: *1234*

תוקף: 5 דקות
_אם לא ביקשת קוד, התעלם מהודעה זו_
```

### reminderJob.js
להוסיף שליחת Push notification ללקוחות עם token רשום, במקביל (לא במקום) ל-WhatsApp.

### timeUtils.js / calendarService.js
לא נדרשים שינויים — הביטול כבר עובד דרך `deleteAppointment`.

## 11. סדר מימוש מוצע (תוספות בלבד)

**שלב א — Backend:**
1. `POST /api/customer/send-otp` + `verify-otp`
2. `sendOTP` ב-whatsappService
3. `GET /api/customer/appointments`
4. `DELETE /api/customer/appointments/:id`
5. טבלאות Firestore: `otp_codes`, `customer_tokens`

**שלב ב — Frontend:**
1. כפתור "התורים שלי" בדף הנחיתה
2. מסך CustomerLogin (טלפון + OTP)
3. מסך MyAppointments (קרובים + היסטוריה + ביטול)
4. InstallPromptBanner (iOS)

**שלב ג — Push:**
1. FCM registration בעת כניסה
2. שליחת push מהשרת (תור חדש, ביטול, תזכורת)
3. בדיקה על אייפון אמיתי
